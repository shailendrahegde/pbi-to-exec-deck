# Example Directory Setup

This document shows the recommended directory structure for using the Claude PowerPoint Constitution.

## Directory Structure

```
C:\Users\[your-username]\claudex\
│
├── Claude PowerPoint Constitution.md    # The governance file (download from repo)
│
├── Example-Storyboard-Analytics.pptx    # Reference example (download from repo)
│
├── [Your-Dashboard].pptx                # Your Power BI export
│   Example: AI-in-One Dashboard - v10.2.pptx
│
├── [Your-Template].pptx                 # Your corporate template (optional)
│   Example: Analytics-Template.pptx
│
└── [Output].pptx                        # Generated by Claude Code
    Example: AI-in-One Dashboard - Executive Analytics.pptx
```

## Step-by-Step Setup

### 1. Create Working Directory

```bash
# Windows
mkdir C:\Users\[your-username]\claudex
cd C:\Users\[your-username]\claudex

# Mac/Linux
mkdir ~/claudex
cd ~/claudex
```

### 2. Download Constitution

- Download `Claude PowerPoint Constitution.md` from this repository
- Save to your working directory

### 3. Export Power BI Dashboard

- Open Power BI → File → Export → PowerPoint
- Save to working directory

### 4. Add Corporate Template (Optional)

- Copy your corporate PowerPoint template to working directory
- Update Section 10 in constitution with file path

### 5. Run Claude Code

Open Claude Code CLI in your working directory:

```bash
cd C:\Users\[your-username]\claudex
```

Run the prompt:

```
"AI-in-One Dashboard - v10.1.pptx" Read this file and convert into an executive ready analytics deck following the Claude PowerPoint Constitution. Copy and paste appropriate screenshots in this new deck. Use Analytics template as a reference guide for visuals and formatting expectations. Enforce section 6 of the constitution. Each page should have compelling insights and a persuasive headline.
```

## File Naming Conventions

**Power BI Export:**
- Use descriptive names: `[Project]-Dashboard-[Version].pptx`
- Include date if relevant: `Q4-Sales-Dashboard-2024-01-15.pptx`

**Corporate Templates:**
- Clear purpose: `Analytics-Template.pptx`, `Strategy-Template.pptx`
- Version if multiple: `Analytics-Template-v2.pptx`

**Output Files:**
- Claude Code typically generates: `[Original-Name] - Executive Analytics.pptx`
- Rename for final distribution: `Q4-Sales-Executive-Brief.pptx`

## Tips

1. **Keep constitution in the same directory** as your dashboard files for easy access
2. **Don't modify the constitution filename** - Claude Code looks for it by name
3. **Use absolute paths** in Section 10 for template references
4. **Clean up temporary files** after generation (*.png, temp_*.pptx)

## Advanced: Multiple Projects

For managing multiple projects:

```
C:\Users\[your-username]\claudex\
│
├── Claude PowerPoint Constitution.md    # Shared constitution
│
├── project-A\
│   ├── Dashboard-A.pptx
│   └── Executive-A.pptx
│
├── project-B\
│   ├── Dashboard-B.pptx
│   └── Executive-B.pptx
│
└── templates\
    ├── Analytics-Template.pptx
    └── Strategy-Template.pptx
```

Update Section 10 paths:
```markdown
"C:\Users\[your-username]\claudex\templates\Analytics-Template.pptx"
```
